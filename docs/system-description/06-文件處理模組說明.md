# 6. 文件處理模組說明

## 主要依據檔案

- `modules/workflow.py`
- `modules/extract_word_chapter.py`
- `modules/extract_word_all_content.py`
- `modules/extract_specific_figure_xml.py`
- `modules/extract_specific_table_xml.py`
- `modules/extract_pdf_img.py`
- `modules/docx_merger.py`
- `modules/docx_provenance.py`
- `app/jobs/executor.py`

## 功能目的

文件處理模組負責依流程或 Mapping 轉換後的步驟執行 Word/PDF 內容擷取、圖片或表格擷取、文字與標題插入、圖片插入、檔案複製、模板合併與後處理。此模組是系統產生交付文件與處理結果的核心。

## 輸入與輸出

輸入來源通常為任務 `files/` 內的 Word、PDF、圖片、資料夾或 Mapping 指定文字。輸出包含 `result.docx`、流程 `log.json`、Mapping 輸出 DOCX、複製後檔案/資料夾與 ZIP 封裝。流程執行結果保存在 `task_store/<task_id>/jobs/<job_id>/`，Mapping 結果保存在 `task_store/<task_id>/mapping_job/<run_id>/`。

## 支援步驟

依 `modules/workflow.py` 的 `SUPPORTED_STEPS`，目前支援擷取 PDF 標籤圖片、擷取 Word 全部內容、擷取 Word 指定章節/標題、插入 Word 指定章節/標題的特定圖片、插入 Word 指定章節/標題的特定表格、插入純文字段落、插入羅馬數字標題、插入項目符號標題、插入阿拉伯數字標題、插入圖片檔、複製檔案與複製資料夾。註解中曾出現但未啟用的步驟不列為已確認功能。

| Step type | 中文標籤 | 主要輸入 | 主要輸出或效果 |
|---|---|---|---|
| `extract_pdf_pages_as_images` | 擷取 PDF 標籤圖片 | PDF 檔、模板插入位置 | 將 PDF 頁面轉為可插入 Word 的圖片內容。 |
| `extract_word_all_content` | 擷取 Word 全部內容 | DOCX、忽略目錄/頁首頁尾設定 | 擷取整份 Word 內容成片段。 |
| `extract_word_chapter` | 擷取 Word 指定章節/標題 | DOCX、章節編號、章節標題、子標題、結束章節 | 擷取指定章節範圍。 |
| `extract_specific_figure_from_word` | 插入特定圖片 | DOCX、章節條件、caption、title、index | 擷取指定 Figure 或表格容器內圖片。 |
| `extract_specific_table_from_word` | 插入特定表格 | DOCX、章節條件、caption、title、index | 擷取指定 Table。 |
| `insert_text` | 插入純文字段落 | 文字、對齊、粗體、字級 | 在輸出文件新增文字段落。 |
| `insert_roman_heading` | 插入羅馬數字標題 | 文字、層級、字級 | 依序產生羅馬數字標題。 |
| `insert_bulleted_heading` | 插入項目符號標題 | 文字、字級 | 新增項目符號標題。 |
| `insert_numbered_heading` | 插入阿拉伯數字標題 | 文字、層級、字級 | 依序產生阿拉伯數字階層標題。 |
| `insert_image` | 插入圖片檔 | 圖片檔、對齊 | 將圖片插入輸出 Word。 |
| `copy_files` | 複製檔案 | 來源、目的資料夾、關鍵字、目標名稱 | 將檔案複製至輸出空間。 |
| `copy_directory` | 複製資料夾 | 來源資料夾、目的資料夾、關鍵字、目標名稱 | 將資料夾或符合條件資料夾複製至輸出空間。 |

## 後處理

流程輸出會進行文件合併、模板套用、來源 provenance 建立與圖表編號後處理。`modules/workflow.py` 中可見圖表 caption 與本文參照重新編號邏輯，包含 Figure、Fig.、圖、Table、Tab.、表等前綴。若後處理失敗，流程 log 會記錄 `postprocess_renumber_figure_table` 的錯誤，但實際回應是否中斷整個流程需依該次執行 log 判讀。

## 處理狀態與例外

背景工作負責包裝流程執行狀態。若任一步驟產生 `status=error`，系統會記錄工作失敗與錯誤摘要。流程亦支援取消檢查，取消時會以 `JobCanceledError` 中斷。常見例外包含來源檔案不存在、PDF ZIP 無效、圖片路徑無效、關鍵字未檢索到檔案或資料夾、未知步驟類型與文件 XML 結構缺失。
