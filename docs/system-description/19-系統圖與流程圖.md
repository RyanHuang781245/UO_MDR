## 任務匯入與來源同步流程

```mermaid
sequenceDiagram
    participant User as 使用者
    participant Web as Flask Web
    participant NAS as NAS Root
    participant DB as DB
    participant FS as task_store
    participant Worker as Worker

    User->>Web: POST /tasks
    Web->>NAS: resolve_nas_path()
    NAS-->>Web: validated source path
    Web->>FS: create task folder, files, output, meta.json
    Web->>DB: insert tasks
    Web->>DB: enqueue task_source_sync job
    Worker->>DB: claim job
    Worker->>NAS: copy source files
    Worker->>FS: write files and update meta.json
    Worker->>DB: complete or fail job
    User->>Web: view task list/status
```

## 流程執行流程

```mermaid
sequenceDiagram
    participant User as 使用者
    participant Web as Flow Routes
    participant DB as DB
    participant Worker as Worker
    participant WF as modules/workflow.py
    participant FS as task_store

    User->>Web: POST /tasks/<task_id>/flows/run
    Web->>Web: validate flow name, output path, steps, template
    Web->>DB: enqueue flow_single job
    Web-->>User: redirect to flow page with job id
    Worker->>DB: claim flow_single
    Worker->>FS: prepare job directory
    Worker->>WF: run_workflow(steps)
    WF->>FS: write result.docx and log.json
    Worker->>DB: update job_records, events, artifacts
    User->>Web: poll /flows/runs/<job_id>/status
```

## Mapping 檢查與執行流程

```mermaid
flowchart TD
    A[上傳或選擇 Mapping Excel] --> B[表頭辨識與列解析]
    B --> C{引用文件檢查}
    C -->|通過| D{擷取條件檢查}
    C -->|失敗| E[status_key=error]
    D -->|通過| F[Mapping Scheme ready]
    D -->|失敗| E
    F --> G[enqueue mapping_scheme_run]
    G --> H[process_mapping_excel]
    H --> I[產生 DOCX / 複製檔案 / ZIP / log]
    I --> J[record mapping_runs]
    E --> K[顯示錯誤 log]
```

## ERD 摘要

```mermaid
erDiagram
    users ||--o| user_roles : has
    roles ||--o{ user_roles : assigned
    tasks ||--o{ job_records : task_id
    job_records ||--o{ job_events : job_id
    job_records ||--o{ job_artifacts : job_id
    tasks ||--o{ mapping_schemes : task_id
    tasks ||--o{ mapping_runs : task_id
    mapping_schemes ||--o{ mapping_runs : scheme_id
    tasks ||--o| task_file_states : task_id
    job_records ||--o| task_execution_locks : job_id
```

備註：除 `users`、`roles`、`user_roles` 外，多數關聯為程式邏輯關聯，ORM 未定義實體外鍵。
